# frontend/src/types/

Shared TypeScript domain types used across pages, components, services,
and the store. One file (`index.ts`) re-exports everything.

## Key types

- **`Goal`** — a single goal with optional `plan: GoalPlan | null` and
  per-goal `planChat: GoalPlanMessage[]`. `goalType` is one of `"big"`,
  `"everyday"`, `"repeating"`.
- **`GoalPlan`** — hierarchical plan: years → months → weeks → days →
  tasks. Generated by `ai/handlers/generateGoalPlan` on the backend.
- **`GoalPlanTask`** — a single leaf task in a plan. Has weight, duration,
  category, completion state.
- **`Task`** — top-level today/tasks-page task (separate from a plan task).
- **`Reminder`** — one-shot or repeating reminder.
- **`CalendarEvent`** — in-app event (NOT a macOS Calendar event — those
  flow through `electron/ipc/device:*` and are kept separate)
- **`MemoryStore`** — `{ facts, preferences, signals, snoozeRecords, taskTimings, lastReflectionAt, reflectionCount }`
- **`HomeChatMessage`** + **`ChatSession`** — home chat history shapes
- **`PendingTask`** — quick-task analysis in flight

## Why these are duplicated server-side

The backend has its own type definitions (mostly inline in route files)
that mirror these. They're kept in sync by hand because the renderer and
backend are separate TypeScript projects with no shared package — and
because the IPC envelope shape is the contract, not the source.

If you change a shape on one side, search the other side for the same
field names and update there too.
